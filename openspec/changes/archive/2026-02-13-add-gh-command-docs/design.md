## Context

当前 `docs/development-environment.md` 文档描述了开发环境所需的各种工具和依赖，但缺少 `gh` (GitHub CLI) 的安装说明。项目的某些技能（如 GitHub PR 审查技能）依赖 `gh` 命令，开发者需要手动安装才能使用这些功能。

## Goals / Non-Goals

**Goals:**
- 在开发环境文档中添加 `gh` 命令的安装说明
- 说明 `gh` 的基本用途和验证方法
- 保持文档简洁，只包含必要信息

**Non-Goals:**
- 不涉及代码变更
- 不涉及架构或功能变更
- 不需要提供详细的 `gh` 使用教程（开发者可参考官方文档）

## Decisions

**文档位置**: 在 `docs/development-environment.md` 中添加 `gh` 安装说明
- 理由：开发环境文档是开发者配置环境的入口，所有必需工具应在此集中说明

**安装方式**: 提供主流操作系统（Linux、macOS、Windows）的安装命令
- 理由：项目可能在不同平台上开发，需要覆盖常见场景

**内容范围**: 仅包含安装和基本验证，不深入使用细节
- 理由：保持文档简洁，详细使用说明可参考 GitHub 官方文档

## Risks / Trade-offs

**风险**: 不同系统版本的安装方式可能有差异
- **缓解**: 提供官方包管理器和官方脚本的安装方式，覆盖大多数情况

**权衡**: 文档简明性与完整性
- 提供 GitHub 官方文档链接，开发者需要时可深入了解
